## What This Is

dtogen is a Go CLI tool that generates Data Transfer Object (DTO) structs and mapper functions from existing Go structs using AST parsing. It supports both single-DTO CLI mode and batch generation via YAML config.

## Commands

```bash
# Build
go build -o dtogen .

# Run tests
go test ./...

# Run single test
go test ./internal/parser/ -run TestParseStruct

# Example CLI usage
./dtogen -s ./testdata -t User -e Password -o user_dto.go

# Example config file usage
./dtogen -c dtogen.yaml
```

## Architecture

**Pipeline:** Config/CLI flags → Parser (AST) → Generator (template) → formatted `.go` file

- **`main.go`** — CLI entry point. Parses flags (`pflag`), delegates to parser and generator. Contains `pascalCaseToSnakeCase` helper for output filenames.
- **`internal/parser/`** — Uses `golang.org/x/tools/go/packages` to load Go source, walks AST to extract `StructInfo` (fields, types, tags, imports). Handles embedded fields.
- **`internal/generator/`** — Filters/renames fields per config, manages import deduplication, renders via embedded `dto.tmpl` template, formats output with `go/format`.
- **`internal/config/`** — YAML config parsing. `GlobalConfig` (output dir, imports, source path) and `DTOConfig` (per-DTO: type, excludes, includes, renames, filters, templates, add_fields).
- **`internal/types/`** — Shared `Filter` type definition.

## Key Patterns

- Generated files are prefixed with `// Code generated by dtogen. DO NOT EDIT.`
- Default DTO naming: `{StructName}DTO`; output filename: snake_case of DTO name + `.go`
- Template (`dto.tmpl`) generates: DTO struct, `ToXDTO` mapper, `ToXDTOList` batch mapper, `IsDTO()` marker method, optional filter logic and added fields
- Import collection is automatic from parsed field types; supports `exclude_imports` to drop unwanted ones
- Config supports `filters` (conditional field transformations) and `add_fields` (extra fields on the DTO)
